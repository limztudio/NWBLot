# Caustics — Design + Plan

## Status
Understand+design pass complete (architecture-mapping workflow). User decisions:
- **Groundwork first**: Phase 0 = the IOR/refraction *material model*; the caustic *producer* is deferred until after Phase 0 (decide then).
- **Richer IOR model**: scalar IOR + transmission color + thickness + a *dedicated refractive-caster flag* (separate from `transparent`).

## Architecture findings (the mapping)
- The RT foundation is ~80% of caustics and directly reusable: per-frame HW TLAS/BLAS + SW GPU-LBVH (static **and** skinned, refit per frame); the per-instance `NwbRtInstanceMaterial` table (instanceID == HW `InstanceID()` == SW leaf); per-mesh index/attribute descriptor arrays ([64]); the per-light importance/slot allocator; the deferred per-light combine point. Caustics = "same foundation, **opposite ray direction, additive deposit**".
- **THE BLOCKER (confirmed by code-read):** there is NO IOR / eta / Fresnel / transmission-direction anywhere. Material schema (`metadata.h` `s_AllowedAssetFields`) allows only `transparent`/`two_sided`. The surface contract (`NwbMeshSurface`) exposes only baseColor/normal/param0/param1/transmittance-tint. Every transparent path — AVBOIT extinction `-log(1-alpha)` and the colored-shadow any-hit — **attenuates** light but never **bends** it.
- Reusable bits already present: `Vector3Refract`/`RefractV` (Snell + total-internal-reflection) in `global/math/vector.h:3043` (pure math, currently unwired); `rt_shadow_plan.md` Ch.2 caustics intent (light-side photon/caustic-map, refract via Snell, additive deposit, colored via Beer-Lambert, dispersion optional).

## Phase 0 — IOR/refraction material groundwork (CURRENT)
Goal: refraction as a first-class refraction model end-to-end — the material classifies (a `refractive` boolean), the shader supplies the values — validated by upgrading the EXISTING colored-shadow absorption to Fresnel + Beer-Lambert. No caustic producer yet. HW + SW in lockstep (mandatory).

### UNIFIED transmittance application (2026-06-24, supersedes the U0-3 engine-side refractive branch)
The shadow transmittance application is now **unified to one path**: the per-material `.surface` hook ALWAYS produces the final per-hit `transmittance`, and the engine applies it ONE way — `payload.transmittance *= nwbShadowDispatchTransmittance(...)` (HW any-hit) / the SW running-transmittance equivalent — with **NO engine-side refractive branch**.
- A refractive material's hook computes its own transmittance by CALLING the engine helper `nwbShadowRefractiveTransmittance(ior, thickness, transmission, incidentDirection, worldNormal)` itself (Fresnel + Beer-Lambert); a plain material returns its flat tint. The hook owns the value either way.
- The shadow-transmittance dispatch returns `float3 nwbShadowDispatchTransmittance(uint shadingModel, NwbShadowHit hit)` — each per-id wrapper returns `nwbShadowSurfaceModel<id>().transmittance`; `default:` returns `float3(1,1,1)`. (This reverts U0-3's "return the whole NwbMeshSurface / engine reads FLAG_REFRACTIVE" form.)
- `NwbMeshSurface` STILL carries `ior`/`thickness`/`transmission` (neutral defaults 1 / 0 / float3(1,1,1)): the surface is the shader's FULL value description — a refractive hook authors all three AND computes its transmittance from them via the helper, storing all four on the surface. The shadow trace reads only `transmittance`; the future caustic producer reads `ior`/`thickness`/`transmission` off this value description.
- `nwbShadowRefractiveTransmittance` is a **shared-contract LIBRARY function**: it lives on `mesh/surface.slangi` (NOT `shadow/shadow_surface.slangi`) so the SAME hook resolves it in BOTH compile contexts — the rasterizer G-buffer pixel shader (which discards transmittance, so it is harmless there) and the shadow trace's transmittance dispatch. Slang built-ins only → byte-identical HW/SW.
- The hook's Fresnel needs the incident direction: `NwbMeshSurfaceInputs.incidentDirection` (+ the `inIncidentDirection` static). The shadow trace fills it from `NwbShadowHit.rayDirection` (the shadow ray's world dir); the rasterizer has NO view buffer in the G-buffer pass, so it supplies a documented dummy (`-normal`) — harmless since the G-buffer discards transmittance.
- `asset.refractive` / `RtInstanceMaterialFlag::Refractive` / `NWB_RT_INSTANCE_MATERIAL_FLAG_REFRACTIVE` STAY end-to-end as the material's refractive **classification** plumbed to the GPU instance record — groundwork for the future caustic producer. The unified shadow trace does **NOT** read `FLAG_REFRACTIVE` (the hook owns transmittance).

### Design decisions (CORRECTED 2026-06-24: material decides only the boolean; the shader decides the values)
- **The material decides only the `refractive` boolean; the shader decides the values.** Same principle as the colored-shadow transmittance (the `.surface` hook already decides the per-hit tint). Only `refractive` (a 0/1 classification flag, exactly like `transparent`) is an authored `asset.*` material field: parsed by `ParseMaterialRenderProperties` → `MaterialCookEntry.refractive` → `Material` (rides the existing `materialFlags` word, **no POD/version change**) → `MaterialSurfaceInfo.refractive` → the RT per-instance occluder record. The refraction VALUES — `ior` (scalar), `thickness` (scalar), `transmission` (Beer-Lambert absorption color) — are **shader-side**: fields the `.surface` hook returns via `NwbMeshSurface`, authored as `.bind` constants. No material POD version bump (MTL4 stays MTL4; the `refractive` flag is a new bit2 in the existing flags word, backward-compatible — old data has bit2=0=not refractive).
- **Engine-generic physics, opt-in via `refractive`.** A `refractive` occluder's per-hit transmittance gets `Fresnel(ior, Schlick)` × `BeerLambert(transmission, thickness/cosθ)` applied by the ENGINE in the shadow any-hit + SW traversal, reading the ior/thickness/transmission the hit's surface hook returned. Non-refractive materials keep current flat-tint behavior → **no visual change to existing demos**. Rationale: Fresnel/Snell is universal physics; the surface supplies the values, the engine owns the physics (mirrors how the engine owns the lighting loop and the material provides the BXDF). Reuses `Vector3Refract`/Schlick.
- **Surface-contract IOR exposure is IN Phase 0 (U0-1).** `NwbMeshSurface` carries `ior`/`thickness`/`transmission` (neutral defaults: 1 / 0 / float3(1,1,1)), set by `nwbMakeMeshSurface` overloads so existing hooks/overloads + the G-buffer packing are unaffected. NOTHING consumes them yet (the shadow-dispatch return + the physics are U0-2/U0-3); this unit only makes the contract able to carry them.

## Deferred — the caustic PRODUCER (choose after Phase 0)
Avoid the rasterized light-view caustic map (a 3rd rendering path that fights the engine's one-abstraction/two-backend discipline). Two viable:
- **Approach 1 — photoreal (light-side photon splat):** true focused crescents; needs a forward light-trace, atomic splat into a black-cleared additive RGBA16F caustic buffer, a denoise/resolve, a NEW HW pipeline (closest-hit + recursion>1 + larger payload), and a SW compute photon kernel; **relaxes** the HW/SW byte-equality bar.
- **Approach 2 — stylized (eye-side refractive gather):** clones the shadow pass's screen-space dispatch + per-slot Texture2DArray; cheap; byte-identical HW/SW; physically *fake* (refractive brightening, not focusing).

Decisions to make then: parity bar (gates Approach 1 vs 2); per-light caustic budget (directional/spot only, 1-4 slots; point lights = omnidirectional = expensive); dispersion (skip v1); additive deposit point (`composite_ps.slang` vs `scene/lighting.slangi`); atomic-splat target (float-image atomics vs R32-packed + resolve pass); the HW per-mesh **position** SRV-array gap (HW shadows omit positions because the BLAS owns them; a refraction closest-hit / receiver-intersection needs explicit positions — the SW path already binds them); non-uniform-instance-scale normal correctness (currently a documented latent divergence in the shadow normal transform — becomes real for refraction).
