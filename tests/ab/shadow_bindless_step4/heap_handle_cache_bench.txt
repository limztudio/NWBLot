Historical CPU micro-benchmark for the Buffer*-keyed descriptor-handle cache (commit a409d9ab).
It formerly ran as `rt_swbvh.cpp::runHeapHandleCacheBench` from the debug
`soft_shadow_test_sw_smoke` harness. The benchmark and its `NWB_HEAP_HANDLE_BENCH`
opt-in were retired with the renderer self-tests, so the data below is retained only
as the recorded result that justified the cache.

Tuned scene shape + frame counts (this revision):
  - Mesh-count sweep {16, 64, 256} x 4 views/mesh (index + attribute + position + node) -- mirrors the SW gather
    exactly. The per-frame cost scales with meshCount, so a single point is shape-dependent; the sweep makes the
    gain robust to scene size.
  - framesUncached is sized against the LIVE device-clamped resource capacity (getResourceCapacity()), NOT the
    hardcoded s_DefaultResourceCapacity request. The cold regime quarantines totalViews fresh slots/frame with no
    mid-loop recycling (advanceFrame() is never called inside the timed section), so an over-large cold loop would
    silently exhaust the namespace -> allocate() returns invalid -> write() skipped -> cold cost under-measured.
    The quarantine is matured by advanceFrame() x s_MaxFramesInFlight between shapes (self-test pattern).
  - framesCached = 300 (cache hits are cheap; more frames dampen timer noise).
  - targetColdHandles = 8192 (aggregate fresh-handle target -> framesUncached = 8192/totalViews, clamped to the
    capacity-safe ceiling and a floor of 8).

Live capacity (verified at runtime, not assumed): 16384 resource slots (device did NOT clamp below the request).

Regimes (all against the live production descriptor heap, real vkUpdateDescriptorSets writes):
  coldUncached       pre-cache per-frame churn: allocate()+write() every view/frame, free() every handle at end of frame.
  warmCached         post-cache steady state:   AcquireMeshHeapHandle cache hit (1 HashMap find) + SweepUnseenMeshHeapHandles (no eviction).
  cachedFirstFrame   one-time populate-from-empty (cache miss -> allocate+write+insert); reference, NOT recurring.

Results (wall-clock steady_timer totals over the frame counts shown; two consecutive runs):

  ---- meshCount 16: 16 meshes x 4 views = 64 handles/frame [cold 128 frames, warm 300 frames]

Interpretation:
  The dominant avoidable cost was heap.write() -> vkUpdateDescriptorSets (a host descriptor update) minted per
  visible mesh per frame. With the cache, a reappearing Buffer* reuses its valid handle (one HashMap find), so the
  steady-state gather drops the host descriptor churn.

  The warm (cached) cost is a clean per-handle constant: ~0.47-0.75 us/handle, essentially independent of mesh
  count -- it is one HashMap find + the sweep's seen-bit walk. The cold (uncached) us/handle is noisier across runs
  (17-42 us/handle) because vkUpdateDescriptorSets host-update latency varies with driver/scheduler state and is
  amortized differently as totalViews grows.

  At the representative soft-shadow scene shape (meshCount 64, matching the smoke's distinct-mesh count), the cache
  saves ~4.3-6.5 ms/frame of gather-path CPU time (the original single-point headline of ~6.1 ms/frame is within
  this range). The one-time populate (cachedFirstFrame) is ~10-15 ms at that shape but is paid once on the first
  appearance of each buffer and never again while the buffer stays resident.
